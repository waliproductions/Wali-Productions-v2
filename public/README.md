# Public assets

Static assets are served from this directory at the site root (e.g.
`/public/favicon.ico` is served at `/favicon.ico`).

## Branding assets referenced by site metadata

The site metadata (`src/app/layout.tsx` and `src/lib/seo.ts`) references the
following files. They are **not** committed yet — add the official or
placeholder artwork here and the metadata will pick them up automatically. No
artwork is generated by the codebase.

| File                     | Used for                          | Recommended size      |
| ------------------------ | --------------------------------- | --------------------- |
| `favicon.ico`            | Browser favicon                   | 32×32 (multi-size ICO)|
| `apple-touch-icon.png`   | Apple Touch Icon (iOS home screen)| 180×180 PNG           |
| `og-image.png`           | Open Graph / Twitter share image  | 1200×630 PNG          |

Until these files are added, browsers and social previews will simply fall back
to defaults; no build error occurs.
